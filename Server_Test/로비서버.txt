로비서버에서는 유저의 id와 pw를 확인한 후 제대로된 유저라면 게임서버의 위치를 가르쳐 주어서
게임서버로의 접속을 허용한다.
그러나 이 시스테을 알게된 해커라면 로비서버의 인증을 건너뛰고 직접 게입서버로 접속할 수 있다.
이러한 상황을 막기 위해서는 로비서버에서 클라이언트로 게임서버 정보를 보낼때,
게임서버로도 [~가 접속할것]이라는 신호를 보낸다면 게임서버는 그와 같은 플레이어만 접속 허용할 수 있다.
여기에 로비서버가 만든 난수를 키로 같이 보낸다면 더욱 확실한 보안이 가능해진다.
DWORD randKey; // 난수 키
위에서 randKey를 난수로 만들어서 클라이언트와 게임서버로 보낸다면 이 키를 가지고 인증을 받은 유저인지
확인할 수 있다.

로비서버에서 게임서버로 접속하는 과정 
1. 클라이언트 ->로비서버 : 게임서버 정보를 달라
2. 로비서버 -> 클라이언트 : 게임서버정보 + 난수키
3. 클라이언트 ->게임서버 접속
4. 클라이언트 -> 게임서버 : 유저ID + 난수키
5. 게임서버 -> 로비서버 : 유저ID + 난수키 인증을 받았는가?
6. 로비서버 -> 게임서버 : 인증 받았는지 여부
7. 게임서버 -> 클라이언트 : 로그인되었는가?
